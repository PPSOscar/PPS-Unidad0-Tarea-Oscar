## ¿En qué consiste la actividad?

La actividad consiste en crear, documentar y publicar un proyecto con Git, MkDocs, GitHub Actions, GitHub Pages y Docker, demostrando un flujo completo de producción segura.

Los puntos importantes de esta actividad son:

1. Crear un repositorio estructurado.
    - Haciendo uso de Git.
2. Escribir la documentación y "transformarla" de manera automática en una web.
    - Utilizando lenguaje Markdown.
3. Automatizar la generación de la documentación.
    -  A través de GitHub Actions.
4. Publicar la documentación.
    - Haciendo uso de GitHub Pages.
5. Replicar el despliegue de manera local a través de un contenedor. 
    - Utilizaré Docker e implantaré la imagen NGINX.

## Objetivos que pretendo alcanzar

- Trabajar el sistema de control de versiones distribuido Git.
- Hacer uso de Markdown.
- Crear _WorkFLow_ en GitHub Actions para generar documentación de manera automática.
- Publicar la web en GitHub Pages.
- Levantar un contenedor con la imagen NGINX para publicar localmente la documentación.
- Documentar el proceso de realización de la actividad.
